fl::CRGB Struct Reference

Detailed Description

Representation of an 8-bit RGB pixel (Red, Green, Blue)

Note
Performance Tip: This struct provides both inline single-pixel methods and references to bulk array operations. For best performance:
  • Single pixels or real-time updates: Use inline methods (setHSV(), nscale8_video(), etc.)
    • Low latency (avoids function call overhead)
    • Good for unpredictable, dynamic patterns
  • Array operations (initialization, effects on ranges): Use bulk functions from fl/fill.h, fl/colorutils.h, and fl/blur.h
    • 2-4x faster than per-pixel inline loops due to loop unrolling and SIMD
    • Examples: fill_rainbow(), nscale8_video(array, count, scale), fadeToBlackBy()
  • See also: fill_rainbow(), fill_gradient<>(), nscale8_video(), fadeLightBy(), fadeToBlackBy(), blur1d(), blur2d() for high-performance batch operations
Examples
Animartrix.ino, Audio.ino, BeatDetection.ino, Blur2d.ino, FxDemoReel100.ino, FxEngine.ino, FxFire2012.ino, FxGfx2Video.ino, FxNoisePlusPalette.ino, FxNoiseRing.ino, FxPacifica.ino, FxPride2015.ino, Overclock.ino, and Pacifica.ino.
