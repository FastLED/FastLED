# Colorimetric strategy harness golden fixtures

This directory holds vendored fixture data the colorimetric strategy harness
reads at host-test time. Every file here is a frozen snapshot generated by an
external tool. Do not hand-edit generated CSV/header contents; regenerate via
the script noted in each file's provenance.

## `verifier_known_bad.csv`

381 rows extracted from the contributor's medium patch-set verifier session
(originally a Google Sheet, snapshot preserved on issue
[#3422](https://github.com/FastLED/FastLED/issues/3422)).

Each row captures one input chromaticity sample driven through the contributor's
LED test rig, the legacy analytical RGBW tuple used for that measurement, and
the colorimeter measurement of the resulting LED.

Important: the `cf_*` columns are a legacy 16-bit `analytical_strict_rgbw16`
snapshot from the verifier sheet. They are not the canonical current FastLED
closed-form reference. The strategy harness computes the live reference from
the checked-out solver and also emits a drift audit comparing that live public
u8 solver against the legacy `cf_*` snapshot.

Rows marked `status=fail` are inputs where the legacy verifier output physically
measured more than the patch's dE budget away from the expected target
chromaticity. That is hardware evidence for the archived snapshot, not proof
that the current live closed-form solver is wrong on the same row.

### Columns

| Column | Type | Meaning |
|---|---|---|
| `patch` | string | Patch name, e.g. `h180_s100_v015` |
| `r_in16`, `g_in16`, `b_in16` | u16 | Source linear RGB input on a 65535-scale |
| `exp_x`, `exp_y` | float | Target CIE 1931 xy chromaticity for this patch |
| `cf_r16`, `cf_g16`, `cf_b16`, `cf_w16` | u16 | Legacy `analytical_strict_rgbw16` RGBW output from the archived verifier sheet |
| `meas_x`, `meas_y`, `meas_Y` | float | Colorimeter measurement when the LED was driven with the legacy `cf_*` values |
| `dE` | float | Perceptual delta-E between `(meas_x, meas_y)` and `(exp_x, exp_y)` for the legacy output |
| `status` | enum | `ok` if that legacy-output dE was within patch budget, `fail` if known-bad, `skip` for degenerate rows |

### How The Harness Uses This

1. Replays each row's `(r_in16, g_in16, b_in16)` through every registered
   strategy after converting the input to the table's u8 domain.
2. Scores strategy accuracy against the live named reference for that row:
   strict closed-form, production policy, layered RGBWW reference, or an
   explicit policy target. It does not treat legacy `cf_*` as the current
   canonical reference.
3. Emits `Reference Fixture Drift Detail` and `Reference Fixture Top Drift Rows`
   so the legacy `cf_*` snapshot can be compared against the current live public
   u8 closed-form surface. This is a staleness/current-surface check, not an
   independent physical proof.
4. Keeps `status=fail` rows visible as hardware-investigation fixtures. A
   strategy that differs from legacy `cf_*` on those rows is only
   candidate-better until it is remeasured on representative LEDs/hardware.

### Regenerating

```bash
uv run python ci/tools/extract_verifier_fixture.py
uv run python ci/tools/gen_colorimetric_fixture_header.py
```

The extractor reads from the verbatim snapshot. To refresh against an upstream
change, first re-fetch the source CSV into the temp staging path the script
expects, or pass `--source` explicitly.
